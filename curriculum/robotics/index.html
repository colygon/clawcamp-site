Control Real Robots with OpenClaw

Bridge AI and physical hardware. Use OpenClaw's Solo CLI to teleoperate robots, record actions, train vision-language models, and deploy learned behaviors to real hardware.

Who It's For
Developers curious about embodied AI and robotics
Value
Hands-on experience with real robot hardware and AI control
Takeaway
A trained robot behavior deployed from demonstration data
Hardware Note

Robot hardware is provided at in-person ClawCamp events. For remote participants, simulation environments are available. Check your local camp's equipment list.

Prerequisites

  • OpenClaw installed (complete Local Install first)
  • Basic Python knowledge
  • Interest in robotics (no prior robotics experience needed)
  • Hardware provided at in-person events

Learning Outcomes

  • Teleoperate a robot via Solo CLI
  • Record demonstration episodes for training
  • Train a VLA (Vision-Language-Action) model on your demos
  • Deploy trained behaviors to real hardware

Step-by-Step Guide

1
Set Up Solo CLI 10 min

Install and configure the Solo CLI — OpenClaw's robotics interface. Solo CLI connects your local environment to robot hardware (or simulation) and provides the teleoperation and recording tools.

Solo CLI works with both real hardware and simulation. If you're working remotely, the simulator provides the same API.
2
Teleoperate 15 min

Take manual control of a robot arm. Learn the control interface — position, grip, speed — and practice executing tasks like pick-and-place, stacking, or sorting. This is how you'll generate training data.

3
Record Episodes 15 min

Capture demonstration data while teleoperating. Each "episode" is a complete task execution — the robot's camera feed, joint positions, and your control inputs are all recorded as training data for the VLA model.

  • Record 10–20 episodes of the same task
  • Vary your approach slightly each time for robustness
  • Review and discard any failed episodes
4
Train VLA Model 20 min

Fine-tune a Vision-Language-Action model on your recorded demonstrations. The VLA model learns to map visual observations and language instructions to robot actions — turning your demonstrations into a generalizable policy.

VLA models learn from surprisingly few demonstrations. 10–20 high-quality episodes is often enough for simple tasks.
5
Deploy to Hardware 15 min

Push the trained model back to the robot. Watch it execute the task autonomously based on what it learned from your demonstrations. Test with variations to see how well the policy generalizes.

  • Load the trained model onto the robot controller
  • Give a natural language task instruction
  • Observe autonomous execution
  • Iterate: record more demos for failure cases
