Deploy Agents on Nebius Serverless

Learn how OpenClaw separates orchestration from inference. Deploy production agents on Nebius infrastructure using Token Factory for pay-per-token GPU access — no GPU rental needed.

Step-by-Step Guide

1
Understand the Architecture 5 min

OpenClaw separates the Gateway (CPU, orchestration) from inference (GPU, Token Factory). The Gateway handles agent logic on cheap CPU compute. Token Factory handles LLM inference on GPU — you pay per token, not per hour.

Key insight: Agent logic is ~20% of the work. Infrastructure is ~80%. This architecture lets you focus on the 20%.
2
Get Your Credentials 5 min

Sign up for Nebius AI Cloud and get your Token Factory API key. You'll need both for the deployment.

  • Create a Nebius AI Cloud account
  • Generate a Token Factory API key from the dashboard
  • Install the Nebius CLI
3
Choose Your Deployment Path 5 min

OpenClaw supports four deployment approaches. This workshop focuses on CPU Serverless (Path 4) for production.

  • Path 1: Local npm + Token Factory (dev/testing)
  • Path 2: Docker container (reproducible)
  • Path 3: GPU Serverless on Nebius (private inference)
  • Path 4: CPU Serverless + Token Factory (production, this workshop)
4
Deploy to Nebius Serverless 15 min

Deploy the pre-built container image to Nebius Serverless. This gives you an always-on agent endpoint with auto-scaling.

Deployment Config

Image: ghcr.io/colygon/openclaw-serverless:latest

Platform: cpu-e2 (or cpu-d3 for EU West)

Resources: 2vCPU / 8GB RAM minimum

Ports: 8080 (health) + 18789 (WebSocket)

Both ports must be explicitly exposed. The EXPOSE directive alone is not sufficient — configure them in the Nebius console.
5
Connect Channels 10 min

Wire up your deployed agent to messaging platforms. OpenClaw supports multi-platform connectivity out of the box.

  • Telegram — via Bot API token
  • WhatsApp — via Business API
  • Discord — via bot token
  • Signal — via Signal CLI
6
Production Hardening 10 min

Secure your deployment for production use. Set up secrets management, health checks, and monitoring.

  • Store secrets with MysteryBox (Nebius secrets manager)
  • Configure health check endpoints
  • Set up basic monitoring and alerting
  • Test failure scenarios and recovery
